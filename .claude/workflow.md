# Development Workflow

## 1. 작업 시작 단계 (Planning)

- 사용자가 새로운 기능을 요구하면 즉시 코딩하지 말고, 먼저 `implementation-plan.md`를 작성하여 승인받는다.
- 기존 코드와의 영향도(Side Effects)를 분석하여 보고한다.

## 2. 구현 단계 (Coding)

- 한 번에 너무 많은 파일을 수정하지 말고, 컴포넌트 단위로 나누어 제안한다.
- 코드를 작성한 후에는 반드시 해당 코드의 로직을 요약 설명한다.

## 3. 검증 단계 (Review & Test)

- 코드를 제출하기 전, `.claude/convention.md`를 준수했는지 스스로 체크리스트를 작성한다.
