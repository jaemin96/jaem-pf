# Coding Conventions

## 1. Version

- `apps/version4` 이외의 버전들은 절대 수정하지 않는다.
- 버전 간의 의존성을 최소화하여 독립적으로 빌드 가능해야 한다.

## 2. Component Pattern

- 정적 사이트이므로 성능 최적화(이미지 최적화, 코드 스플리팅)를 최우선으로 한다.
- 모든 텍스트 데이터는 해당 기능 모듈의 `data/` 폴더에 분리하여 관리하여 다국어 지원이나 내용 수정이 용이하게 한다. (예: `module/portfolio/data/en.ts`, `kr.ts`)
- 테마 변경 스타일은 항상 `styles/` 경로에서 관리하며 이를 `src/app/global.css` 에서 유동적으로 불러와서 사용할 수 있게 작업한다.
- 반복되는 코드나 UI들은 항상 컴포넌트화하며 커스터마이징 요소들은 무조건 분리된 컴포넌트 내에서 독립적으로 시행한다 (이 변경사항이 기존 기능에 영향을 주지 말아야함)
- 스타일 변경 작업이 기존 다른 스타일 경험에 영향이 가지않도록 연관된 스타일도 함께 수정 반영한다.

## 3. Critical Rule

- **절대 과거 버전의 파일을 직접 수정하지 마라.**
- **허락없이 추가적인 패키지 설치는 지양하라.**
